# === Cross-Draft Validation ===
# Demonstrates automatic constraint re-validation when the calendar changes
# after a draft was created.
#
# Scenario: We create two weekly drafts (week 3 and week 4), generate both,
# then commit week 3. When we re-open week 4, the system detects that the
# calendar changed and re-validates all assignments. Workers who worked the
# Apr 25-26 weekend now violate the alternating-weekends rule for the
# May 2-3 weekend — those assignments are automatically removed.

# Pin Marco to grill on Saturday midday (grill hours: 10-17)
pin marco grill saturday midday

# Create two concurrent drafts (non-overlapping)
draft create 2026-04-20 2026-04-26 --force
draft create 2026-04-27 2026-05-03 --force
draft list

# Generate both drafts — scheduler assigns workers to all shifts including weekends
draft generate 2
draft generate 3

# Commit week 3 to the calendar — locks in weekend assignments for Apr 25-26
draft commit 2 week 3 with Saturday grill

# Re-open week 4 draft — system detects calendar changed, re-validates,
# and removes weekend assignments for workers who worked Apr 25-26
draft open 3

# === Freeze Line ===
# Demonstrates the freeze-line guardrail for historical calendar dates.
#
# The freeze line defaults to yesterday. Dates on or before
# the freeze line are "frozen" — creating a draft that covers them
# triggers a warning. The intended workflow for intentional history edits:
# 1. Check freeze status
# 2. Explicitly unfreeze the historical dates
# 3. Create and commit the draft (no warning since dates are unfrozen)
# 4. Unfreezes auto-clear after committing historical dates

# Discard the cross-validation demo draft so we start clean
draft discard 3

# Check the current freeze line — no unfreezes active yet
calendar freeze-status

# Try to create a draft covering historical dates — blocked by freeze line
draft create 2026-04-06 2026-04-12

# Override with --force to create the draft despite frozen dates
draft create 2026-04-06 2026-04-12 --force

# Generate and commit the corrected schedule
draft generate
draft commit 4 corrected week 1 (forced)

# Auto-refreeze: all temporary unfreezes cleared after committing
# a draft that included historical dates
calendar freeze-status

# Now demonstrate the unfreeze workflow as an alternative to --force
# Unfreeze week 1 dates — only frozen portion is affected
calendar unfreeze 2026-04-06 2026-04-12

# Verify the unfreeze is active
calendar freeze-status

# Create a draft covering week 1 — no freeze error because we unfroze
draft create 2026-04-06 2026-04-12

# Generate and commit
draft generate
draft commit 5 corrected week 1 (unfrozen)

# Auto-refreeze clears all unfreezes after committing historical dates
calendar freeze-status

# === What-If Exploration ===
# Demonstrates the hint system for exploring schedule changes
# within a draft session without modifying any real data.
#
# Scenario: We create a draft for week 4 and generate a schedule.
# Then we explore hypothetical changes:
#   - What if Nina could work the grill? (grant-skill)
#   - What if we closed the pizza station on Monday? (close-station)
# Each hint shows a diff of the schedule impact. We can list,
# revert, and re-explore freely — nothing is persisted.

# Create and generate a draft for week 4
draft create 2026-04-27 2026-05-03 --force
draft generate 6

# Explore: what if Nina (beverage/register/sandwich) could also do grill?
what-if grant-skill nina grill

# Explore: what if we close the pizza station on Monday at 11:00?
what-if close-station pizza 2026-04-27 11

# Review all active hints
what-if list

# Undo the station closure (keep the skill grant)
what-if revert

# Check the list again — only the skill grant remains
what-if list

# Clear all remaining hints to restore the original schedule
what-if revert-all

# Discard the draft — we were just exploring
draft discard 6
